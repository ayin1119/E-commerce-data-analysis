# 电商购买概率模型评估总报告

## 1. 基础信息

| 项目         | 内容                                                                                                  |
| ------------ | ----------------------------------------------------------------------------------------------------- |
| 项目名称     | 电商用户-商品未来 7 天购买概率预测                                                                    |
| 任务类型     | 二分类任务，同时也是排序任务                                                                          |
| 预测目标     | 判断某个用户在未来 7 天是否会购买某个商品                                                             |
| 目标字段     | `label_buy_7d`，其中 1 表示未来 7 天购买，0 表示未来 7 天未购买                                     |
| 输出概率     | 逻辑回归输出`buy_probability`，LightGBM 输出 `lgbm_probability`，XGBoost 输出 `xgb_probability` |
| 主要模型     | SGD Logistic Regression、LightGBM、XGBoost                                                            |
| 最终候选模型 | XGBoost 全特征版本和 LightGBM 全特征版本                                                              |
| 主要应用场景 | 推荐排序、营销触达候选池、用户-商品购买概率预估                                                       |

本报告按照“模型评估报告”的要求进行整理，覆盖基础信息、数据集描述、评估指标、模型性能分析、误差分析、过拟合/欠拟合讨论，以及环境与部署考量。

## 2. 数据集描述

### 2.1 输入特征

项目最初使用 65 字段特征宽表，特征主要分为四类：

| 特征类型          | 说明                                                   |
| ----------------- | ------------------------------------------------------ |
| 用户-商品交叉特征 | 用户对某个商品的浏览、收藏、加购、购买、最近行为间隔等 |
| 用户整体特征      | 用户 7 天内总行为数、购买数、活跃天数、行为商品数等    |
| 商品整体特征      | 商品浏览数、购买数、转化率、活跃用户数等               |
| 用户-品类交叉特征 | 用户对某个品类的行为次数、购买次数、行为占比等         |

后续在 65 字段基础上新增了 3 个行为路径特征，形成 68 字段宽表：

| 新增字段                          | 含义                     |
| --------------------------------- | ------------------------ |
| `pre_buy_behavior_count`        | 用户购买前发生的行为数量 |
| `purchase_count`                | 用户历史购买次数         |
| `pre_buy_behavior_per_purchase` | 平均每次购买前的行为次数 |

实际进入模型的数值型特征为 62 个，因为 `dataset_type`、`snapshot_date`、`user_id`、`item_id`、`item_category`、`label_buy_7d` 不进入模型训练。

### 2.2 数据集划分和标签分布

后续特征工程、LightGBM 和 XGBoost 模型主要基于 1:10 版本的 68 字段宽表。该版本数据集分布如下：

| 数据集 |  样本数 | 正样本数 | 负样本数 | 正样本率 | 负/正比例 |
| ------ | ------: | -------: | -------: | -------: | --------: |
| test   |  86,039 |    7,781 |   78,258 |    9.04% |   10.06:1 |
| train  |  30,248 |    2,735 |   27,513 |    9.04% |   10.06:1 |
| valid  |  27,605 |    2,525 |   25,080 |    9.15% |    9.93:1 |
| 合计   | 143,892 |   13,041 |  130,851 |    9.06% |   10.03:1 |

可以看到，1:10 版本整体正样本率约为 9.06%，比原始真实比例更均衡，适合模型调试、特征筛选和模型结构对比。但最终业务解释时仍需注意，真实线上购买率会低于该采样版本。

### 2.3 不同正负样本比例版本

为了观察采样比例对模型效果的影响，前期构建了原始比例、1:50、1:20、1:10 和 1:5 多个版本。结果如下：

| 版本        |    样本数 | 字段数 | 正样本率 | Test AUC | Test PR-AUC | Test Top10%购买率 | Test Top10%召回率 | 说明               |
| ----------- | --------: | -----: | -------: | -------: | ----------: | ----------------: | ----------------: | ------------------ |
| 原始比例 LR | 3,238,318 |     65 |    0.40% |   0.8625 |      0.0558 |             4.51% |            66.38% | 最接近真实业务分布 |
| 1:50 LR     |   665,091 |     65 |    1.96% |   0.8876 |      0.1402 |            13.26% |            67.60% | 更接近真实分布     |
| 1:20 LR     |   273,861 |     65 |    4.76% |   0.8692 |      0.3252 |            31.13% |            65.36% | 中间比例对照       |
| 1:10 LR     |   143,892 |     65 |    9.06% |   0.8880 |      0.4636 |            51.31% |            56.73% | 后续特征工程主版本 |
| 1:5 LR      |    78,246 |     65 |   16.67% |   0.8917 |      0.6734 |            75.69% |            45.41% | 快速实验版本       |

这些版本不能简单横向比较 PR-AUC 和 Top 购买率，因为不同版本的正样本率不同。1:5 和 1:10 的评估集更均衡，所以 PR-AUC 和 Top 购买率看起来更高；原始比例最接近真实业务，但正样本率极低，因此 PR-AUC 数值较低。最终业务评估必须参考原始比例版本，而模型调试可以使用 1:10 或 1:20 版本。

## 3. 评估指标选择

本任务是购买预测，正样本相对稀少。如果只看准确率，模型即使大量预测为未购买，也可能得到很高准确率，但这种结果没有业务价值。因此，本项目不把准确率作为核心指标，而重点使用以下指标：

| 指标                               | 作用                                                         |
| ---------------------------------- | ------------------------------------------------------------ |
| AUC                                | 衡量模型整体排序能力，判断正样本是否排在负样本前面           |
| PR-AUC                             | 更关注正样本识别能力，适合不平衡二分类任务                   |
| Top5% / Top10% / Top20% 真实购买率 | 衡量模型高分样本中真实购买比例，直接对应推荐和营销候选池质量 |
| Top 比例召回率                     | 衡量高分候选池能覆盖多少真实购买样本                         |
| Brier Score / Log Loss / ECE       | 衡量预测概率是否可信，用于概率校准评估                       |
| 训练时间 / 模型大小 / 推理速度     | 衡量工程部署可行性                                           |

其中，排序场景优先看 AUC、PR-AUC 和 Top10% 指标；概率解释场景需要进一步看 Brier Score、Log Loss 和 ECE。

## 4. 模型迭代过程与性能结果

### 4.1 行为路径特征加入后的效果

原 65 字段模型已经能识别高购买概率样本，但没有显式描述“购买之前经历了多少行为”。加入行为路径特征后，模型效果明显提升：

| 对比项            | 原 65 字段 LR | 新 68 字段 LR |           变化 |
| ----------------- | ------------: | ------------: | -------------: |
| Valid AUC         |        0.8552 |        0.8773 |        +0.0221 |
| Valid PR-AUC      |        0.4075 |        0.4636 |        +0.0561 |
| Test AUC          |        0.8880 |        0.8986 |        +0.0105 |
| Test PR-AUC       |        0.4636 |        0.4866 |        +0.0230 |
| Test Top10%购买率 |        51.31% |        53.59% | +2.28 个百分点 |
| Test Top10%召回率 |        56.73% |        59.25% | +2.52 个百分点 |

行为路径特征加入后，test PR-AUC 从 0.4636 提升到 0.4866，test Top10% 购买率从 51.31% 提升到 53.59%。这说明行为路径不是冗余字段，而是补充了用户从行为到购买的路径强度。

### 4.2 特征筛选结果

在 68 字段版本基础上，继续进行特征筛选。结果如下：

| 实验版本            | 特征数 | Valid AUC | Valid PR-AUC | Test AUC | Test PR-AUC | Test Top10%购买率 | Test Top10%召回率 | 训练耗时 |
| ------------------- | -----: | --------: | -----------: | -------: | ----------: | ----------------: | ----------------: | -------: |
| `top20_features`  |     20 |    0.8704 |       0.4826 |   0.9014 |      0.5463 |            55.20% |            61.03% |    0.20s |
| `top30_features`  |     30 |    0.8786 |       0.4849 |   0.9048 |      0.5406 |            55.31% |            61.15% |    0.37s |
| `top40_features`  |     40 |    0.8841 |       0.4956 |   0.9060 |      0.5389 |            54.76% |            60.54% |    0.53s |
| `all_68_features` |     62 |    0.8773 |       0.4636 |   0.8986 |      0.4866 |            53.59% |            59.25% |    0.79s |
| `top50_features`  |     50 |    0.8835 |       0.4661 |   0.9020 |      0.4728 |            52.55% |            58.10% |    0.58s |

Top20 特征版本表现最好，test AUC 为 0.9014，test PR-AUC 为 0.5463，test Top10% 真实购买率为 55.20%。相比全特征线性模型，Top20 特征更少、训练更快、效果更好。这说明在线性模型中，部分低重要性特征可能引入噪声。

在完成特征筛选后，进一步尝试非线性模型。模型对比结果如下：

| 排名 | 模型                   | 实验版本                               | 特征数 | Valid AUC | Valid PR-AUC | Test AUC | Test PR-AUC | Test Top5%购买率 | Test Top10%购买率 | Test Top10%召回率 | 训练耗时 |
| ---: | ---------------------- | -------------------------------------- | -----: | --------: | -----------: | -------: | ----------: | ---------------: | ----------------: | ----------------: | -------: |

|    2 | LightGBM               | `lgbm_top20_no_weight`               |     20 |    0.9042 |       0.5640 |   0.9228 |      0.6416 |           75.52% |            58.67% |            64.86% |    0.76s |
|    3 | XGBoost                | `xgb_top20_scale_pos_weight`         |     20 |    0.9033 |       0.5583 |   0.9226 |      0.6394 |           74.89% |            58.69% |            64.89% |    0.79s |
|    4 | LightGBM               | `lgbm_all_features_no_weight`        |     62 |    0.9045 |       0.5644 |   0.9235 |      0.6393 |           75.61% |            58.63% |            64.82% |    0.90s |
|    5 | XGBoost                | `xgb_top20_no_weight`                |     20 |    0.9039 |       0.5616 |   0.9224 |      0.6387 |           75.49% |            58.83% |            65.04% |    0.76s |
|    6 | XGBoost                | `xgb_all_features_scale_pos_weight`  |     62 |    0.9037 |       0.5621 |   0.9221 |      0.6380 |           75.08% |            58.51% |            64.70% |    0.93s |
|    7 | LightGBM               | `lgbm_top20_scale_pos_weight`        |     20 |    0.9039 |       0.5601 |   0.9227 |      0.6356 |           75.01% |            58.94% |            65.17% |    0.71s |
|    8 | LightGBM               | `lgbm_all_features_scale_pos_weight` |     62 |    0.9056 |       0.5673 |   0.9232 |      0.6348 |           75.08% |            58.79% |            65.00% |    0.92s |
|    9 | SGD_LogisticRegression | `lr_top20_features`                  |     20 |    0.8704 |       0.4826 |   0.9014 |      0.5463 |           64.85% |            55.20% |            61.03% |    0.20s |

从表中可以看出，树模型整体明显优于 SGD 逻辑回归。LR Top20 的 test PR-AUC 为 0.5463，而 XGBoost 和 LightGBM 的 test PR-AUC 基本达到 0.63 以上。树模型能够学习用户行为、商品转化、行为路径之间的非线性组合关系，因此在正样本识别和 Top 样本召回方面更强。

如果严格按照 valid PR-AUC 选择模型，LightGBM 的 `lgbm_all_features_scale_pos_weight` 表现最好，valid PR-AUC 为 0.5673，test PR-AUC 为 0.6348。XGBoost 的 `xgb_all_features_scale_pos_weight` 是按 valid PR-AUC 选出的 XGBoost 最优版本，test PR-AUC 为 0.6380。若只看 test PR-AUC，`xgb_all_features_no_weight` 最高，test PR-AUC 为 0.6435，test Top10% 购买率为 58.99%。

综合来看，XGBoost 和 LightGBM 都可以作为最终候选模型。若强调测试集排序效果，XGBoost 全特征版本略优；若强调按验证集选型的稳定性，LightGBM 和 XGBoost 加权版本都可以保留。

### 4.4 概率校准结果

LightGBM 原始概率存在明显偏高问题，因此做了 Platt Scaling 和 Isotonic Regression 校准。test 集结果如下：

| 方法         |    AUC | PR-AUC | Brier Score | Log Loss |    ECE | 平均预测概率 | 真实购买率 |
| ------------ | -----: | -----: | ----------: | -------: | -----: | -----------: | ---------: |
| `original` | 0.9232 | 0.6348 |      0.1101 |   0.3547 | 0.1884 |       27.88% |      9.04% |
| `platt`    | 0.9232 | 0.6348 |      0.0490 |   0.1718 | 0.0104 |        9.80% |      9.04% |
| `isotonic` | 0.9228 | 0.6197 |      0.0492 |   0.1749 | 0.0110 |        9.82% |      9.04% |

校准前，LightGBM 原始平均预测概率为 27.88%，而真实购买率为 9.04%，说明原始概率明显偏高。Platt 校准后平均预测概率为 9.80%，更接近真实购买率，同时 AUC 和 PR-AUC 不变。说明 Platt 校准适合作为业务概率解释版本。

需要注意，XGBoost 当前已经完成排序模型评估，但尚未完成概率校准。如果最终选择 XGBoost 作为主模型，建议继续补做 XGBoost 的 Platt / Isotonic 校准。

## 5. 高方差 / 高偏差与过拟合讨论

### 5.1 当前证据

从 valid 和 test 指标看，树模型没有出现明显的过拟合。以 LightGBM 和 XGBoost 为例，valid AUC 约为 0.904-0.906，test AUC 约为 0.922-0.924，test 指标反而更高。这说明当前模型并不是典型的训练集过拟合、测试集崩溃的情况。

不过，test 指标高于 valid 也可能说明不同时间窗口的数据分布存在差异。例如 test 标签窗口可能处于购买行为更集中的阶段，因此 test 更容易识别。这属于时间切分场景中的分布差异，需要在后续更多时间窗口上验证稳定性。

### 5.2 学习曲线说明

当前实验没有按训练样本比例生成标准学习曲线，因此不能严格画出“训练样本量增加时 train/valid loss 如何变化”的曲线。本报告用特征数量变化和模型复杂度变化做初步替代分析。

特征数量变化结果显示，Top20 线性模型优于全特征线性模型，说明对于线性模型而言，减少冗余特征可以降低噪声。模型复杂度变化结果显示，XGBoost 和 LightGBM 明显优于 LR，说明当前任务不是单纯高方差问题，而是线性模型表达能力不足，非线性模型能捕捉更多组合模式。

后续如果需要严格满足学习曲线要求，建议用 20%、40%、60%、80%、100% 训练样本分别训练同一模型，并记录 train PR-AUC 和 valid PR-AUC。

## 7. 特征重要性与业务解释

在树模型中，重要特征集中在用户-商品交叉行为和行为路径相关特征。XGBoost 的前几类重要信号包括：

| 特征类型          | 代表字段                                                                                        | 业务含义                                   |
| ----------------- | ----------------------------------------------------------------------------------------------- | ------------------------------------------ |
| 用户-商品行为强度 | `user_item_cart_count_7d`, `user_item_total_behavior_count_7d`, `user_item_view_count_7d` | 用户对具体商品的兴趣越强，购买概率越高     |
| 最近行为距离      | `user_item_last_cart_days`, `user_item_last_behavior_days`                                  | 最近发生关键行为的用户更可能购买           |
| 行为路径特征      | `pre_buy_behavior_per_purchase`, `purchase_count`                                           | 用户历史购买路径和购买强度影响未来购买     |
| 商品转化能力      | `item_view_to_buy_rate_7d`, `item_total_behavior_count_7d`                                  | 商品自身转化能力影响购买概率               |
| 用户兴趣分散程度  | `user_behavior_item_count_7d`, `user_category_behavior_item_count_7d`                       | 用户浏览范围越广，单个商品购买概率可能下降 |

这说明模型不是单纯依赖商品热度，而是综合考虑用户是否对该商品有行为、用户是否有购买习惯、商品是否容易转化，以及用户兴趣是否集中。

| XGBoost 批量预测速度   | 约 1,075,000 样本/秒 | 在当前 sandbox 环境中，对 10,000 条 test 样本批量预测的测量结果    |
| XGBoost 单样本平均耗时 |   约 0.00093 ms/样本 | 批量预测平均值，不等同于线上单条请求延迟                           |
| LR Top20 模型大小      |            0.0032 MB | 线性模型，体积最小                                                 |
| LightGBM 模型大小      |            0.2815 MB | 树模型，体积较小                                                   |
| XGBoost 模型大小       |            0.1949 MB | 树模型，体积较小                                                   |
| 预测资源需求           |             CPU 即可 | 当前实验未使用 GPU                                                 |
| 内存需求               |                 中低 | 143,892 行、68 字段版本可直接加载；原始比例 3,238,318 行需关注内存 |

从工程角度看，LR 最轻量、解释性最好；LightGBM 和 XGBoost 的模型体积也很小，但表达能力更强。当前 XGBoost 批量预测速度在实验环境中很快，适合离线批量打分。线上实时服务中，实际延迟还会受到接口、数据读取、特征生成和网络开销影响，因此不能只看模型本身预测耗时。

## 9. 最终模型选择建议

综合模型效果和工程可用性，本报告给出两个层面的建议：

| 场景                       | 推荐模型                         | 推荐概率字段               | 原因                                         |
| -------------------------- | -------------------------------- | -------------------------- | -------------------------------------------- |
| 推荐排序、Top 用户商品筛选 | XGBoost 全特征版本               | `xgb_probability`        | test PR-AUC 和 Top10% 指标最高，排序能力最强 |
| 稳定业务概率解释           | LightGBM + Platt 校准            | `platt_lgbm_probability` | 校准后平均预测概率接近真实购买率，解释更稳   |
| 简单可解释基线             | LR Top20                         | `buy_probability`        | 特征少、模型小、解释性强，但效果低于树模型   |
| 接近真实业务比例验证       | 原始比例 LR 或后续原始比例树模型 | Top 排序结果               | 原始比例最接近真实线上分布                   |

如果只选择一个最终排序模型，建议优先选择 XGBoost 全特征版本作为主模型候选。如果报告需要强调“不使用 test 集选模型”，则建议保留按 valid PR-AUC 选择的 XGBoost 加权版本或 LightGBM 加权版本作为稳健候选。无论选择哪个树模型，如果要输出真实购买概率，都建议继续做概率校准。

## 10. 当前不足与后续工作

当前报告已经覆盖了不同采样比例、行为路径特征、特征筛选、LightGBM、XGBoost 和 LightGBM 概率校准。但仍有以下不足：

| 不足                       | 后续改进                                                  |
| -------------------------- | --------------------------------------------------------- |
| 没有标准训练样本学习曲线   | 按 20%、40%、60%、80%、100% 训练样本生成 train/valid 曲线 |
| 原始比例版本暂未训练树模型 | 在原始比例宽表上尝试 LightGBM / XGBoost                   |
| 错误分析只抽取少量典型样本 | 后续按品类、用户活跃度、行为路径类型分层分析              |
| 时间稳定性验证不足         | 增加更多 snapshot_date 或滚动时间窗口验证                 |

## 11. 总结

本项目从原始 65 字段逻辑回归模型开始，依次完成了样本比例实验、行为路径特征扩展、特征筛选、LightGBM、概率校准和 XGBoost 模型实验。实验结果表明，新增行为路径特征可以显著提升模型效果，Top20 特征在线性模型中表现最好，而非线性树模型进一步提升了正样本识别能力。

从最终模型对比看，XGBoost 和 LightGBM 都明显优于逻辑回归。其中，XGBoost 全特征版本在 test PR-AUC 和 Top10% 指标上略优，适合作为最终推荐排序模型候选；LightGBM 经过 Platt 校准后概率更接近真实购买率，适合业务概率解释。整体来看，项目已经从“能训练模型”推进到“可以比较不同模型、选择最终候选模型并考虑部署”的阶段。
