# 1:5 正负样本比例购买概率模型报告

## 1. 报告背景

本报告针对 1:5 正负样本比例版本的购买概率模型进行总结。该版本在原始正负样本比例宽表的基础上进行负样本抽样，保留全部正样本，并在 train、valid、test 各数据集中分别抽取正样本数量 5 倍的负样本，形成正负样本比例为 1:5 的建模数据集。

本模型目标是预测用户未来 7 天是否会购买某个商品，目标字段为 `label_buy_7d`。其中 `label_buy_7d = 1` 表示未来 7 天发生购买，`label_buy_7d = 0` 表示未来 7 天未发生购买。模型输出为 `buy_probability`，即用户对该商品的未来购买概率。

1:5 版本的主要作用是作为 1:10 版本和原始比例版本之间的中间实验版本。相比原始比例版本，1:5 数据更均衡，模型更容易学习正样本特征；相比 1:10 版本，1:5 保留的负样本更少，训练速度更快，但也可能降低对真实负样本分布的覆盖。

## 2. 数据集说明

本次 1:5 特征宽表共有 78,246 行、65 个字段，包含 train、valid、test 三个数据集。样本分布如下：

| 数据集 | 总样本数 | 正样本数 | 负样本数 | 正样本率 | 负/正比例 |
| ------ | -------: | -------: | -------: | -------: | --------: |
| test   |   46,686 |    7,781 |   38,905 |   16.67% |    5.00:1 |
| train  |   16,410 |    2,735 |   13,675 |   16.67% |    5.00:1 |
| valid  |   15,150 |    2,525 |   12,625 |   16.67% |    5.00:1 |
| 合计   |   78,246 |   13,041 |   65,205 |   16.67% |    5.00:1 |

## 3. 建模流程

本次建模流程使用已经封装好的购买概率模型 pipeline，主要步骤包括：

```text
读取 1:5 特征宽表
→ 检查必要字段
→ 自动选择数值型特征
→ 排除 ID、日期、标签等非建模字段
→ 缺失值填充
→ 标准化
→ 使用 SGD 逻辑回归训练模型
→ 输出购买概率
→ 计算 AUC、PR-AUC
→ 计算 Top 比例命中效果
→ 输出特征重要性和预测结果
```

模型使用 `SGDClassifier(loss="log_loss")` 实现逻辑回归形式的购买概率模型。该方法适合较大规模样本，因为每轮训练主要是线性扫描样本，训练复杂度相对可控。同时模型配置了 `class_weight="balanced"`，用于进一步缓解正负样本不均衡问题。

## 4. 模型整体评估结果

验证集和测试集的整体评估指标如下：

| 数据集 | 样本数 | 正样本数 | 正样本率 |    AUC | PR-AUC |
| ------ | -----: | -------: | -------: | -----: | -----: |
| valid  | 15,150 |    2,525 |   16.67% | 0.8533 | 0.5923 |
| test   | 46,686 |    7,781 |   16.67% | 0.8917 | 0.6734 |

从结果看，valid AUC 为 0.8533，test AUC 为 0.8917，说明模型具有较好的区分正负样本能力。PR-AUC 在 valid 和 test 上分别为 0.5923 和 0.6734，明显高于随机情况下的正样本率，说明模型能够有效识别购买概率较高的样本。

由于 1:5 版本正样本率为 16.67%，比原始自然比例版本更高，因此 PR-AUC 和 Top 比例真实购买率通常会比原始比例版本更高。这并不代表业务真实场景一定有同样高的购买率，而是说明在经过采样后的训练与评估数据中，模型具备较好的排序和识别能力。

## 5. Top 比例评估结果

在实际业务中，更重要的不是单纯判断所有样本是否购买，而是找出模型认为最可能购买的一部分用户-商品组合。因此本报告重点查看 Top 5%、Top 10%、Top 20% 样本中的真实购买率和覆盖真实购买样本比例。

| 数据集 | Top 比例 | Top 样本数 | Top 真实购买率 | 覆盖真实购买样本比例 | 相对整体购买率提升 |
| ------ | -------: | ---------: | -------------: | -------------------: | -----------------: |
| valid  |    5.00% |        757 |         70.67% |               21.19% |            4.24 倍 |
| valid  |   10.00% |      1,515 |         69.17% |               41.50% |            4.15 倍 |
| valid  |   20.00% |      3,030 |         56.27% |               67.52% |            3.38 倍 |
| test   |    5.00% |      2,334 |         81.23% |               24.37% |            4.87 倍 |
| test   |   10.00% |      4,668 |         75.69% |               45.41% |            4.54 倍 |
| test   |   20.00% |      9,337 |         61.33% |               73.59% |            3.68 倍 |

从 Top 比例结果看，test 集 Top 5% 样本真实购买率达到 81.23%，相比 test 集整体正样本率提升 4.87 倍。test 集 Top 10% 样本真实购买率为 75.69%，覆盖了 45.41% 的真实购买样本。test 集 Top 20% 样本真实购买率为 61.33%，覆盖了 73.59% 的真实购买样本。

这说明模型能够把大量真实购买样本集中到较高预测概率区间，适合用于候选商品排序、用户推荐和营销触达场景。

## 7. 时间复杂度分析

设训练样本数为 `n`，验证集或测试集样本数为 `m`，特征数量为 `d`，模型迭代轮数为 `T`，TopK 样本数量为 `k`。

本次建模流程的主要时间复杂度如下：

| 步骤             |       时间复杂度 | 说明                                   |
| ---------------- | ---------------: | -------------------------------------- |
| 读取宽表         |     `O(n * d)` | 需要读取所有样本和特征                 |
| 特征数值转换     |     `O(n * d)` | 将特征转换为可建模的数值格式           |
| 缺失值填充       |     `O(n * d)` | 使用中位数填充缺失值                   |
| 标准化           |     `O(n * d)` | 对特征进行标准化处理                   |
| SGD 逻辑回归训练 | `O(T * n * d)` | 每轮训练线性扫描样本                   |
| 预测概率         |     `O(m * d)` | 对验证集和测试集输出购买概率           |
| AUC / PR-AUC     |   `O(m log m)` | 精确计算 AUC、PR-AUC 需要排序          |
| Top 比例评估     |   `O(m log k)` | 使用`nlargest` 取 TopK，避免完整排序 |

本版本数据量为 78,246 行，相比原始比例版本大幅减少，因此训练成本明显低于原始比例版本。训练阶段主要复杂度为：

```text
O(T * n * d)
```

其中 `n` 是训练集样本数，`d` 是特征数，`T` 是迭代轮数。本次 pipeline 使用 SGD 逻辑回归，训练过程接近线性扫描样本，适合这种宽表建模任务。

Top 比例评估使用 `nlargest` 获取概率最高的样本，而不是对全部预测结果完整排序，因此 TopK 评估复杂度由完整排序的 `O(m log m)` 优化为：

```text
O(m log k)
```

此外，特征转为 `float32` 可降低内存占用，空间上由近似 `O(n * d * 8)` 降为约 `O(n * d * 4)`。

## 8. 与其他样本比例版本的关系

1:5 版本、1:10 版本和原始比例版本各有作用：

| 版本         | 特点                                            | 适用场景                             |
| ------------ | ----------------------------------------------- | ------------------------------------ |
| 1:5 版本     | 样本更均衡，训练速度快，正样本信号更明显        | 快速实验、验证模型流程、初步特征筛选 |
| 1:10 版本    | 比 1:5 保留更多负样本，兼顾训练效率和负样本覆盖 | 模型调参、稳定性对比                 |
| 原始比例版本 | 最接近真实业务分布，但样本极度不平衡            | 最终业务评估、真实场景验证           |

1:5 版本的结果通常会比原始比例版本看起来更好，尤其是 PR-AUC 和 Top 真实购买率，因为评估集中正样本占比更高。因此，1:5 版本更适合用于模型开发阶段，而原始比例版本更适合用于最终业务判断。

## 9. 结果解读与思考

本次 1:5 版本模型表现较好，test AUC 达到 0.8917，test PR-AUC 达到 0.6734。Top 比例结果也说明模型能够较好地把真实购买样本排到高概率区间。

从业务角度看，模型已经能够初步回答“哪些用户-商品组合更可能发生购买”这个问题。后续可以将预测概率用于商品推荐候选排序、用户触达优先级排序、重点商品曝光策略等场景。
