# 新增行为路径特征后的购买概率模型报告

## 1. 报告背景

本报告针对新增行为路径特征后的购买概率模型进行总结。原模型使用 65 字段特征宽表，本次在原宽表基础上新增了 3 个行为路径相关特征，形成 68 字段版本的新特征宽表，并重新训练购买概率模型。

本模型目标仍然是预测用户未来 7 天是否会购买某个商品，目标字段为 `label_buy_7d`。其中，`label_buy_7d = 1` 表示未来 7 天发生购买，`label_buy_7d = 0` 表示未来 7 天未发生购买。模型输出字段为 `buy_probability`，表示用户对该商品的未来购买概率。

## 2. 新增特征说明

本次新增了 3 个行为路径特征：

| 新增字段                          | 含义                     | 作用                           |
| --------------------------------- | ------------------------ | ------------------------------ |
| `pre_buy_behavior_count`        | 用户购买前发生的行为数量 | 反映用户购买前的行为积累程度   |
| `purchase_count`                | 用户购买次数             | 反映用户历史购买强度           |
| `pre_buy_behavior_per_purchase` | 平均每次购买前的行为次数 | 反映用户从行为到购买的路径长度 |

原来的 65 字段主要描述用户、商品、用户-商品、用户-品类在 7 天观察窗口内的行为次数、比例、活跃天数和最近行为距离。本次新增的行为路径特征补充了“购买之前行为路径长度”这一信息，使模型不仅能看到行为数量，还能看到用户从浏览、收藏、加购等前置行为到购买之间的路径强度。

## 3. 数据集说明

本次新版特征宽表共有 143,892 行、68 个字段，包含 train、valid、test 三个数据集。样本分布如下：

| 数据集 | 总样本数 | 正样本数 | 负样本数 | 正样本率 | 负/正比例 |
| ------ | -------: | -------: | -------: | -------: | --------: |
| test   |   86,039 |    7,781 |   78,258 |    9.04% |   10.06:1 |
| train  |   30,248 |    2,735 |   27,513 |    9.04% |   10.06:1 |
| valid  |   27,605 |    2,525 |   25,080 |    9.15% |    9.93:1 |
| 合计   |  143,892 |   13,041 |  130,851 |    9.06% |   10.03:1 |

## 4. 建模流程

本次建模流程如下：

```text
读取 68 字段新版特征宽表
→ 检查 dataset_type、label_buy_7d 和新增路径特征是否存在
→ 自动选择数值型特征
→ 排除 ID、日期、标签等非建模字段
→ 缺失值填充
→ 特征标准化
→ 使用 SGD 逻辑回归训练模型
→ 输出购买概率
→ 计算 AUC、PR-AUC
→ 计算 Top 比例命中效果
→ 输出特征重要性
```

模型仍然使用 `SGDClassifier(loss="log_loss")`，也就是基于随机梯度下降的逻辑回归模型。这样做的好处是可以保持模型结构不变，只观察“新增行为路径特征”对模型效果的影响，避免因为模型更换导致无法判断提升来源。

## 5. 模型整体评估结果

验证集和测试集的整体评估指标如下：

| 数据集 | 样本数 | 正样本数 | 正样本率 |    AUC | PR-AUC |
| ------ | -----: | -------: | -------: | -----: | -----: |
| valid  | 27,605 |    2,525 |    9.15% | 0.8773 | 0.4636 |
| test   | 86,039 |    7,781 |    9.04% | 0.8986 | 0.4866 |

从结果看，新版模型在 valid 集上的 AUC 为 0.8773，PR-AUC 为 0.4636；在 test 集上的 AUC 为 0.8986，PR-AUC 为 0.4866。这说明模型具有较好的排序能力，并且在识别真实购买样本方面表现较稳定。

## 9. 时间复杂度分析

设训练样本数为 `n`，验证集或测试集样本数为 `m`，特征数量为 `d`，模型迭代轮数为 `T`，TopK 样本数量为 `k`。

原模型特征数量为 65，本次新版模型特征数量为 68，只增加了 3 个字段。因此，整体复杂度形式没有变化，只是特征维度 `d` 略微增加。

| 步骤             |       时间复杂度 | 说明                                   |
| ---------------- | ---------------: | -------------------------------------- |
| 读取宽表         |     `O(n * d)` | 需要读取所有样本和特征                 |
| 特征数值转换     |     `O(n * d)` | 将特征转换为可建模的数值格式           |
| 缺失值填充       |     `O(n * d)` | 使用中位数填充缺失值                   |
| 标准化           |     `O(n * d)` | 对特征进行标准化处理                   |
| SGD 逻辑回归训练 | `O(T * n * d)` | 每轮训练线性扫描样本                   |
| 预测概率         |     `O(m * d)` | 对验证集和测试集输出购买概率           |
| AUC / PR-AUC     |   `O(m log m)` | 精确计算 AUC、PR-AUC 需要排序          |
| Top 比例评估     |   `O(m log k)` | 使用`nlargest` 取 TopK，避免完整排序 |

由于本次只新增 3 个特征，`d` 从 65 增加到 68，训练时间和内存占用只会小幅增加。与模型效果提升相比，这个计算成本是可以接受的。

## 10. 结果解读与思考

本次实验说明，行为路径特征对购买概率预测是有帮助的。原来的 65 字段版本已经包含了大量行为次数、比例和最近行为距离特征，但这些特征更多是在描述“发生了多少行为”。新增路径特征进一步补充了“购买前经历了多少行为”和“用户历史购买路径是否短”的信息，因此能够提升模型对购买意愿的识别能力。

从特征重要性看，`purchase_count`、`pre_buy_behavior_count`、`pre_buy_behavior_per_purchase` 都进入了重要特征区间，说明模型确实使用了这些新增字段，而不是简单忽略它们。这也验证了前面“先加入行为路径特征再调参”的判断是合理的。

## 11. 总结

本次完成了新增行为路径特征后的购买概率模型训练与评估。新版宽表从原来的 65 个字段增加到 68 个字段，新增了 `pre_buy_behavior_count`、`purchase_count`、`pre_buy_behavior_per_purchase` 三个行为路径特征。模型在 test 集上的 AUC 达到 0.8986，PR-AUC 达到 0.4866，Top 10% 样本真实购买率达到 53.59%，并覆盖 59.25% 的真实购买样本。

整体来看，行为路径特征带来了有效提升，建议将 68 字段版本作为后续模型调试的新基线，再继续进行概率校准、特征筛选和模型结构优化。
