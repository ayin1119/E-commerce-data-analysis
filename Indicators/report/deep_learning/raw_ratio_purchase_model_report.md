# 原始正负样本比例购买概率模型报告

## 1. 报告背景

本报告针对“正负样本未作抽样处理”的购买概率模型进行总结。该版本不再使用经过 1:10 采样处理后的训练集，而是使用从原始行为表重新生成的自然比例特征宽表进行建模。模型目标是预测用户在未来 7 天内是否会购买某个商品，目标字段为 `label_buy_7d`。

本次建模使用的宽表为原始比例版本，整体样本保留了真实业务中的正负样本分布，因此它比 1:10 采样版本更接近实际线上预测场景。由于真实购买行为在全部用户-商品样本中占比很低，因此本版本更关注模型的排序能力、Top 样本命中能力以及 PR-AUC 等适合不平衡样本的指标。

## 2. 数据版本说明

本次原始比例宽表由原始行为表重新生成，不使用之前的错误宽表，也不对负样本进行抽样。宽表包含 train、valid、test 三个数据集，字段数量为 65 个，目标标签为 `label_buy_7d`。

数据集整体情况如下：

| 数据集 |    样本数 | 正样本数 |  负样本数 | 正样本率 |
| ------ | --------: | -------: | --------: | -------: |
| train  | 1,030,893 |    2,735 | 1,028,158 |    0.27% |
| valid  | 1,061,515 |    2,525 | 1,058,990 |    0.24% |
| test   | 1,145,910 |    7,781 | 1,138,129 |    0.68% |
| 合计   | 3,238,318 |   13,041 | 3,225,277 |    0.40% |

从样本分布可以看出，原始比例版本存在明显的类别不平衡问题。整体上，负样本远多于正样本，整体正负样本比例约为 1:247。因此，本模型不能只看准确率，而应重点关注 AUC、PR-AUC、Top 比例真实购买率和召回率。

## 3. 模型方案

本次模型使用购买概率模型，输出字段为 `buy_probability`，表示用户未来 7 天购买该商品的概率。模型训练流程已经封装为 pipeline，主要步骤包括读取数据、自动选择数值特征、排除 ID 和标签字段、缺失值填充、标准化、模型训练、预测概率、评估指标计算、Top 比例验证和特征重要性输出。

模型采用 SGD 逻辑回归，即 `SGDClassifier(loss="log_loss")`。该模型适合较大规模样本训练，因为它每轮训练主要是对样本进行线性扫描，相比普通批量优化方式更适合当前百万级样本宽表。同时，配置中使用了 `class_weight="balanced"`，用于缓解原始比例样本中正负样本极度不平衡的问题。

## 4. 模型整体评估结果

验证集和测试集结果如下：

| 数据集 |    样本数 | 正样本数 | 正样本率 |    AUC | PR-AUC |
| ------ | --------: | -------: | -------: | -----: | -----: |
| valid  | 1,061,515 |    2,525 |    0.24% | 0.8393 | 0.0192 |
| test   | 1,145,910 |    7,781 |    0.68% | 0.8625 | 0.0558 |


从结果看，valid AUC 为 0.8393，test AUC 为 0.8625，说明模型对购买样本和未购买样本具有较好的排序区分能力。由于原始比例版本正样本占比极低，PR-AUC 的数值不会很高，这是不平衡业务场景下的正常现象。相比整体购买率，模型在 Top 区间内能够显著提升真实购买样本密度，因此更适合作为用户-商品推荐、营销触达和候选排序模型使用。

## 5. Top 比例评估结果

在真实业务中，通常不会对所有用户-商品组合都采取动作，而是优先关注模型预测概率最高的一部分样本。因此，本报告重点观察 Top 5%、Top 10%、Top 20% 样本中的真实购买率和覆盖真实购买样本比例。

| 数据集 | Top 比例 | Top 样本数 | Top 真实购买率 | 覆盖真实购买样本比例 | 相对整体购买率提升 |
| ------ | -------: | ---------: | -------------: | -------------------: | -----------------: |
| valid  |    5.00% |     53,075 |          1.91% |               40.20% |            8.04 倍 |
| valid  |   10.00% |    106,151 |          1.45% |               61.15% |            6.11 倍 |
| valid  |   20.00% |    212,303 |          0.91% |               76.44% |            3.82 倍 |
| test   |    5.00% |     57,295 |          6.20% |               45.64% |            9.13 倍 |
| test   |   10.00% |    114,591 |          4.51% |               66.38% |            6.64 倍 |
| test   |   20.00% |    229,182 |          2.74% |               80.58% |            4.03 倍 |


从 Top 比例结果看，test 集 Top 5% 样本真实购买率为 6.20%，相比 test 整体购买率 0.68%，提升约 9.13 倍；test 集 Top 10% 样本真实购买率为 4.51%，覆盖了 66.38% 的真实购买样本；test 集 Top 20% 样本真实购买率为 2.74%，覆盖了 80.58% 的真实购买样本。

这说明模型虽然面对非常不平衡的原始样本，但仍然能够把大量真实购买样本排到较高概率区间。对于业务使用来说，可以优先关注预测概率最高的 Top 10% 或 Top 20% 用户-商品组合。

## 7. 时间复杂度分析

设训练样本数为 `n`，验证集或测试集样本数为 `m`，特征数量为 `d`，模型迭代轮数为 `T`，TopK 评估样本数为 `k`。

本次建模流程的主要时间复杂度如下：

| 步骤             |       时间复杂度 | 说明                                   |
| ---------------- | ---------------: | -------------------------------------- |
| 读取宽表         |     `O(n * d)` | 需要读取所有样本和特征                 |
| 特征数值转换     |     `O(n * d)` | 对数值特征进行类型转换                 |
| 缺失值填充       |     `O(n * d)` | 使用中位数填充缺失值                   |
| 标准化           |     `O(n * d)` | 对所有特征进行标准化                   |
| SGD 逻辑回归训练 | `O(T * n * d)` | 每轮训练线性扫描样本                   |
| 预测概率         |     `O(m * d)` | 对验证集和测试集进行预测               |
| AUC / PR-AUC     |   `O(m log m)` | 精确 AUC 与 PR-AUC 需要按概率排序      |
| Top 比例评估     |   `O(m log k)` | 使用`nlargest` 取 TopK，避免完整排序 |

本次主要的时间复杂度优化体现在三个方面。第一，模型采用 SGD 逻辑回归，使训练复杂度接近 `O(T * n * d)`，适合百万级样本。第二，在 Top 比例评估中使用 `nlargest`，只取概率最高的 TopK 样本，将完整排序的 `O(m log m)` 优化为 TopK 选择的 `O(m log k)`。第三，将特征转为 `float32`，降低内存占用，空间上由近似 `O(n * d * 8)` 降为约 `O(n * d * 4)`。

需要注意，AUC 和 PR-AUC 本身属于排序型指标，因此在精确计算时仍然会包含 `O(m log m)` 的复杂度。这部分是指标计算本身的成本，不是模型训练的主要瓶颈。

## 8. 结果解读与思考

原始比例版本更接近真实业务场景，但样本不平衡程度更严重。整体正样本率只有 0.40%，因此如果使用普通准确率，模型即使全部预测为负样本也会得到很高准确率，这没有实际意义。本项目选择 AUC、PR-AUC 和 Top 比例指标，是更适合当前任务的评估方式。

从 AUC 看，模型在 valid 和 test 上都保持在 0.84 以上，说明排序能力较稳定。从 Top 比例看，模型能够把真实购买样本集中到预测概率较高的区域。尤其是 test Top 10% 能覆盖 66.38% 的真实购买样本，说明该模型已经可以作为后续推荐排序或营销触达的基础版本。

不过，当前模型仍有进一步提升空间。由于原始比例版本极度不平衡，PR-AUC 仍然较低，说明模型在高精度识别正样本方面还可以继续优化。后续可以继续尝试特征筛选、特征交叉、非线性模型、样本权重调节、阈值策略优化，以及对不同时间窗口结果进行稳定性验证。

## 9. 与 1:10 采样版本的关系

1:10 版本适合模型快速训练和调试，因为正负样本比例更均衡，模型更容易学习到购买样本特征。但 1:10 版本并不完全反映真实业务分布。原始比例版本虽然训练难度更大，但更接近真实预测场景，因此更适合用于最终业务评估。
