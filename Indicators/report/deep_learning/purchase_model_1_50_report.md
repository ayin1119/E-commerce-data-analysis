# 1:50 正负样本比例购买概率模型报告

## 1. 报告背景

本报告针对 1:50 正负样本比例版本的购买概率模型进行总结。该版本在已有特征宽表基础上保留全部正样本，并按正样本数量的 50 倍保留负样本，形成 1:50 的训练、验证和测试数据集。

本模型目标是预测用户未来 7 天是否会购买某个商品，目标字段为 `label_buy_7d`。其中，`label_buy_7d = 1` 表示未来 7 天发生购买，`label_buy_7d = 0` 表示未来 7 天未发生购买。模型输出为 `buy_probability`，表示用户对该商品的未来购买概率。

## 2. 数据集说明

本次 1:50 特征宽表共有 665,091 行、65 个字段，包含 train、valid、test 三个数据集。样本分布如下：

| 数据集 | 总样本数 | 正样本数 | 负样本数 | 正样本率 | 负/正比例 |
| ------ | -------: | -------: | -------: | -------: | --------: |
| test   |  396,831 |    7,781 |  389,050 |    1.96% |   50.00:1 |
| train  |  139,485 |    2,735 |  136,750 |    1.96% |   50.00:1 |
| valid  |  128,775 |    2,525 |  126,250 |    1.96% |   50.00:1 |
| 合计   |  665,091 |   13,041 |  652,050 |    1.96% |   50.00:1 |


可以看到，每个数据集均保持 1:50 的正负样本比例，整体正样本率为 1.96%。这说明该版本比 1:20 更接近真实业务的不平衡状态，同时仍然比原始自然比例版本更容易训练。

## 3. 建模流程

本次建模继续使用封装后的购买概率模型 pipeline，主要流程包括：

```text
读取 1:50 特征宽表
→ 检查必要字段
→ 自动选择数值型特征
→ 排除 ID、日期、标签等非建模字段
→ 缺失值填充
→ 标准化
→ 使用 SGD 逻辑回归训练模型
→ 输出购买概率
→ 计算 AUC、PR-AUC
→ 计算 Top 比例命中效果
→ 输出特征重要性和预测结果
```

模型使用 `SGDClassifier(loss="log_loss")` 实现逻辑回归形式的购买概率模型。该模型适合较大规模样本训练，因为每轮训练主要对样本进行线性扫描，训练复杂度相对可控。同时，模型配置了 `class_weight="balanced"`，用于缓解 1:50 样本中仍然存在的正负样本不平衡问题。

## 4. 模型整体评估结果

验证集和测试集的整体评估指标如下：

| 数据集 |  样本数 | 正样本数 | 正样本率 |    AUC | PR-AUC |
| ------ | ------: | -------: | -------: | -----: | -----: |
| valid  | 128,775 |    2,525 |    1.96% | 0.8688 | 0.1519 |
| test   | 396,831 |    7,781 |    1.96% | 0.8876 | 0.1402 |


从结果看，valid AUC 为 0.8688，test AUC 为 0.8876，说明模型仍然具有较好的正负样本排序能力。valid PR-AUC 为 0.1519，test PR-AUC 为 0.1402。由于 1:50 版本的正样本率仅为 1.96%，PR-AUC 相比 1:5、1:10、1:20 版本下降是正常现象，因为评估集中的正样本密度更低。

整体来看，1:50 模型更接近真实业务分布下的模型表现，适合作为原始比例版本之前的重要对照实验。

## 5. Top 比例评估结果

在实际业务中，更重要的是从大量用户-商品组合中找出购买概率最高的一部分样本。因此，本报告重点观察 Top 5%、Top 10%、Top 20% 样本中的真实购买率和真实购买样本覆盖率。

| 数据集 | Top 比例 | Top 样本数 | Top 真实购买率 | 覆盖真实购买样本比例 | 相对整体购买率提升 |
| ------ | -------: | ---------: | -------------: | -------------------: | -----------------: |
| valid  |    5.00% |      6,438 |         17.80% |               45.39% |            9.08 倍 |
| valid  |   10.00% |     12,877 |         12.81% |               65.31% |            6.53 倍 |
| valid  |   20.00% |     25,755 |          7.78% |               79.41% |            3.97 倍 |
| test   |    5.00% |     19,841 |         17.66% |               45.02% |            9.00 倍 |
| test   |   10.00% |     39,683 |         13.26% |               67.60% |            6.76 倍 |
| test   |   20.00% |     79,366 |          8.21% |               83.72% |            4.19 倍 |


从 Top 比例结果看，test 集 Top 5% 样本真实购买率达到 17.66%，相对 test 集整体购买率提升 9.00 倍；test 集 Top 10% 样本真实购买率为 13.26%，覆盖了 67.60% 的真实购买样本；test 集 Top 20% 样本真实购买率为 8.21%，覆盖了 83.72% 的真实购买样本。

这说明模型能够有效地将真实购买样本集中到较高预测概率区间。对于实际推荐、候选排序或营销触达场景，可以优先关注 Top 10% 或 Top 20% 样本。

需要注意的是，逻辑回归系数会受到特征相关性的影响，因此特征解释需要结合业务理解，而不能孤立看单个系数。

## 7. 时间复杂度分析

设训练样本数为 `n`，验证集或测试集样本数为 `m`，特征数量为 `d`，模型迭代轮数为 `T`，TopK 样本数量为 `k`。

本次建模流程的主要时间复杂度如下：

| 步骤             |       时间复杂度 | 说明                                   |
| ---------------- | ---------------: | -------------------------------------- |
| 读取宽表         |     `O(n * d)` | 需要读取所有样本和特征                 |
| 特征数值转换     |     `O(n * d)` | 将特征转换为可建模的数值格式           |
| 缺失值填充       |     `O(n * d)` | 使用中位数填充缺失值                   |
| 标准化           |     `O(n * d)` | 对特征进行标准化处理                   |
| SGD 逻辑回归训练 | `O(T * n * d)` | 每轮训练线性扫描样本                   |
| 预测概率         |     `O(m * d)` | 对验证集和测试集输出购买概率           |
| AUC / PR-AUC     |   `O(m log m)` | 精确计算 AUC、PR-AUC 需要排序          |
| Top 比例评估     |   `O(m log k)` | 使用`nlargest` 取 TopK，避免完整排序 |

本版本数据量为 665,091 行，比 1:20 更大，但仍明显小于原始自然比例版本。训练阶段主要复杂度为：

```text
O(T * n * d)
```

其中 `n` 是训练集样本数，`d` 是特征数，`T` 是迭代轮数。本次 pipeline 使用 SGD 逻辑回归，训练过程接近线性扫描样本，适合当前大样本宽表建模任务。

Top 比例评估使用 `nlargest` 获取概率最高的 TopK 样本，而不是对全部预测结果完整排序，因此 TopK 评估复杂度由完整排序的：

```text
O(m log m)
```

优化为：

```text
O(m log k)
```

此外，特征转为 `float32` 可以降低内存占用，使空间占用由近似 `O(n * d * 8)` 降为约 `O(n * d * 4)`。

## 8. 与其他样本比例版本的关系

1:5、1:10、1:20、1:50 和原始比例版本各有作用：

| 版本         | 特点                                     | 适用场景                             |
| ------------ | ---------------------------------------- | ------------------------------------ |
| 1:5 版本     | 样本更均衡，训练速度快，正样本信号更明显 | 快速实验、模型流程验证、初步特征筛选 |
| 1:10 版本    | 训练效率和负样本覆盖相对折中             | 调参、特征筛选、对比实验             |
| 1:20 版本    | 保留更多负样本，更接近真实负样本分布     | 稳定性验证、采样比例对比             |
| 1:50 版本    | 负样本覆盖进一步增加，更接近原始分布     | 接近真实场景的模型评估               |
| 原始比例版本 | 最接近真实业务分布，但样本极度不平衡     | 最终业务验证和汇报                   |

1:50 版本的意义在于，它比 1:20 更接近真实业务场景，同时仍然比原始比例版本更可控。后续可以将 1:50 作为接近真实比例的对照版本，用于观察随着负样本增加，模型排序能力和 Top 命中效果是否仍然稳定。

## 9. 结果解读与思考

本次 1:50 版本模型在 test 集上 AUC 达到 0.8876，说明在更不平衡的样本环境下，模型仍然具备较好的排序能力。与更均衡的 1:5、1:10 版本相比，1:50 的 PR-AUC 和 Top 真实购买率通常会有所下降，这是因为正样本密度降低，不代表模型一定退化。
